# 덤프링 모바일 앱 소스코드 구조 및 용도 가이드

본 문서는 덤프링(Dumpring) 모바일 앱 프로젝트(`dumpring_app`) 내부의 주요 Dart 파일 목록과 각 파일의 세부 용도 및 역할을 설명합니다.

---

## 1. 진입점 (Entry Point)
* **main.dart**
  - **용도**: 앱의 최초 기동 지점입니다. 
  - **역할**: 전역 테마 설정을 로드하고 첫 화면으로 로그인 화면(login_screen.dart)을 배치하여 실행합니다.

---

## 2. 화면 구성 (Screens - `lib/screens/`)
각 사용자 역할(기사, 차주, 지주, 관리자)별 UI 화면과 로직이 모여 있습니다.

### 2.1. 인증 및 사용자 정보
* **login_screen.dart**
  - **용도**: 로그인 화면
  - **역할**: 로그인 처리 및 유저의 가입 상태에 따라 대기 화면, 문서 업로드 화면, 혹은 메인 홈 화면으로 분기 처리합니다.
* **role_selection_screen.dart**
  - **용도**: 역할 선택 화면
  - **역할**: 회원가입 시 기사/차주/지주 등 어떤 유형의 사용자로 가입할지 선택합니다.
* **register_screen.dart**
  - **용도**: 기사 및 차주 회원가입 화면
  - **역할**: 기사 또는 차주로 가입하기 위한 기본적인 폼 입력창을 렌더링하고 가입 API를 요청합니다.
* **site_register_screen.dart**
  - **용도**: 현장 담당자 회원가입 화면
  - **역할**: 상차 현장 관리자 및 작업자 가입을 담당합니다.
* **drop_off_register_screen.dart**
  - **용도**: 사토장 지주 회원가입 화면
  - **역할**: 사토장 관리자/지주 가입을 담당합니다.
* **profile_edit_screen.dart**
  - **용도**: 내 정보 수정 화면
  - **역할**: 사용자의 이름, 비밀번호 변경 등 프로필 편집 기능을 지원합니다.

### 2.2. 기사(Driver) 업무 화면
* **driver_home_screen.dart**
  - **용도**: 기사 메인 홈 화면
  - **역할**: 상단의 내 배차 정보 현황(진행 중 운행)을 실시간으로 확인하고, 하단 필터를 통해 새로운 배차 공고를 조회 및 수락합니다.
* **driver_dispatch_confirm_screen.dart**
  - **용도**: 배차 상세 및 취소/수락 관리 화면
  - **역할**: 수락하려는 혹은 수락 완료된 배차 오더의 출발지/도착지 위치, 운임료, 예상 거리를 확인하고 수락/취소/운행을 제어합니다.
* **driver_meter_screen.dart**
  - **용도**: GPS 운행 미터기 화면
  - **역할**: 주행 시작 버튼을 눌러 GPS 수집을 가동하고 목적지 도착 시 운행을 완료합니다.
* **driver_history_screen.dart**
  - **용도**: 기사 정산 및 운송 내역 화면
  - **역할**: 과거에 완료했거나 정산된 운송 실적 데이터를 리스트로 열람합니다.
* **driver_pending_screen.dart**
  - **용도**: 기사 가입 심사 대기 화면
  - **역할**: 필수 면허 서류 등이 관리자의 최종 승인을 대기하고 있는 상태임을 유저에게 알립니다.
* **driver_document_upload_screen.dart**
  - **용도**: 기사 서류 제출 화면
  - **역할**: 화물운송종사자격증, 운전면허증, 통장 사본 등의 사진을 서버로 전송합니다.

### 2.3. 차주(Owner) 업무 화면
* **owner_home_screen.dart**
  - **용도**: 차주 홈 화면
  - **역할**: 소속 기사의 목록 정보와 현재 보유 및 운행 중인 차량 목록을 모니터링하고 관리합니다.
* **owner_pending_screen.dart**
  - **용도**: 차주 가입 승인 대기 화면
  - **역할**: 관리자의 서류 심사를 대기 중인 상태를 표시합니다.
* **owner_document_upload_screen.dart**
  - **용도**: 차주 증빙 서류 제출 화면
  - **역할**: 사업자등록증, 통장 사본 등 가입 관련 필수 서류를 업로드합니다.

### 2.4. 사토장 지주(Drop-off) 화면
* **drop_off_home_screen.dart**
  - **용도**: 사토장 지주 홈 화면
  - **역할**: 하차지에 도달한 기사의 덤프트럭 정보를 검수하여 토사가 정상 반입되었음을 최종 승인(APPROVED)하거나 반려(REJECTED) 처리합니다.

### 2.5. 프레임 및 관리자
* **main_home_frame.dart**
  - **용도**: 다중 역할 뷰 배분 프레임
  - **역할**: 사용자의 회원 정보 내 역할 세팅에 맞추어 `DriverHomeScreen`, `OwnerHomeScreen` 등 알맞은 홈 스크린으로 자동 라우팅 브릿지 처리를 수행합니다.
* **admin_home_screen.dart**
  - **용도**: 가입 심사자용 관리 화면
  - **역할**: 가입 신청을 낸 회원들의 프로필 및 제출 서류들을 검수하고 정식 승인을 내려주는 간이 백오피스 페이지입니다.
* **common_drawer.dart**
  - **용도**: 전역 왼쪽 드로어 메뉴 바
  - **역할**: 내 프로필 바로가기, 로그아웃, 회원 탈퇴 등의 네비게이션을 담고 있습니다.
* **dashboard_screen.dart**
  - **용도**: 운송 대시보드
  - **역할**: 임시로 구축된 성과/실적 도표 뷰어입니다.
* **sdui_screen.dart**
  - **용도**: 동적 UI(SDUI) 화면
  - **역할**: 서버에서 지정한 UI 형식을 그대로 불러와서 로드하는 테스트 스크린입니다.

---

## 3. 동적 UI 엔진 (Server-Driven UI - `lib/sdui/`)
* **sdui_service.dart**
  - **용도**: SDUI 데이터 연결 및 관리 클래스
  - **역할**: API 통신으로 UI 템플릿과 스타일 사양을 땡겨오고 로컬 캐싱을 적용합니다.
* **sdui_renderer.dart**
  - **용도**: SDUI UI 위젯 해석기
  - **역할**: 서버에서 보내준 UI JSON 메타데이터를 실제 Flutter의 Material 위젯 요소로 직접 실시간 변환하여 빌드해 주는 파서입니다.
* **sdui_model.dart**
  - **용도**: 동적 위젯 데이터 규격 모델 정의
* **sdui_theme_model.dart**
  - **용도**: 동적 테마 구성 요소 관리 모델 정의
* **driver_overlay_meter.dart**
  - **용도**: 주행 오버레이 위젯 구성 파일

---

## 4. 공통 설정 및 공유 위젯 (Shared - `lib/shared/`)
* **app_config.dart**
  - **용도**: 앱 전역 설정 변수 파일
  - **역할**: 백엔드 서버 엔드포인트 URL(`baseUrl` 및 `wsBaseUrl`)을 기재하여 연동하며, 개발 시 포트 수정 시 이 파일만 수정하면 됩니다.
* **dr_scaffold.dart**
  - **용도**: 공통 프리미엄 다크테마 레이아웃 프레임 위젯
  - **역할**: 앱의 통일성 있는 디자인 디자인 시스템 및 배경 레이아웃을 제공합니다.
