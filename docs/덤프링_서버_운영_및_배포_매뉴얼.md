# 덤프링 (Dumpring) 통합 기동 및 배포 매뉴얼

본 문서는 덤프링(Dumpring) 플랫폼의 **로컬 개발 환경 기동 방법(FastAPI, Next.js, Flutter)**, **데이터베이스 마이그레이션 및 기초 데이터 설정(Supabase & Alembic)**, 그리고 **운영 서버(Render) 배포 절차**를 다루는 통합 실무 매뉴얼입니다.

---

## 1. 시스템 아키텍처 및 환경 정보

*   **백엔드 API**: Python 3.11 + FastAPI + SQLAlchemy + Alembic
*   **웹 프론트엔드**: Next.js 16 (React 19, TypeScript, TailwindCSS v4)
*   **모바일 앱**: Flutter (Android / iOS / Web)
*   **데이터베이스**: Supabase PostgreSQL (AWS Seoul 리전)
*   **운영 호스팅**: Render (FastAPI Web Service)

---

## 2. 로컬 개발 환경 기동 (Local Run Guide)

### 2.1. 백엔드 (FastAPI) 기동

1.  **가상환경 활성화**:
    프로젝트 최상위 루트 디렉토리에서 터미널을 열고 가상환경을 기동합니다.
    ```bash
    # Windows (PowerShell)
    .\venv\Scripts\Activate.ps1

    # Linux / macOS
    source venv/bin/activate
    ```

2.  **의존성 패키지 설치**:
    ```bash
    pip install -r requirements.txt
    ```

3.  **환경 변수 설정 (`.env`)**:
    루트 디렉토리에 `.env` 파일을 생성하고 아래 정보를 입력합니다.
    ```ini
    PROJECT_NAME="덤프링 (dumpring)"
    SECRET_KEY="SUPER_SECRET_KEY_REPLACE_THIS_IN_PRODUCTION"
    DATABASE_URL="postgresql+asyncpg://postgres.vvdydhxmwdrrazwyjvvu:crypto.co.kr@aws-1-ap-northeast-2.pooler.supabase.com:6543/postgres"
    ```
    > [!IMPORTANT]
    > Supabase 연결 시 커넥션 풀을 정상적으로 사용하기 위해 포트는 반드시 **6543**을 지정해야 합니다.

4.  **서버 실행**:
    ```bash
    uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
    ```
    *   **Swagger API 문서**: [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

---

### 2.2. 웹 프론트엔드 (Next.js - `dumpring_web`) 기동

1.  **웹 프로젝트 디렉토리로 이동 및 패키지 설치**:
    ```bash
    cd dumpring_web
    npm install
    ```

2.  **개발 서버 실행**:
    ```bash
    npm run dev
    ```
    *   **웹 로컬 주소**: [http://localhost:3000](http://localhost:3000)

---

### 2.3. 모바일 앱 (Flutter - `dumpring_app`) 기동

1.  **앱 프로젝트 디렉토리로 이동 및 의존성 패키지 설치**:
    ```bash
    cd dumpring_app
    flutter pub get
    ```

2.  **시뮬레이터/디바이스 연결 및 실행**:
    *   **연결 기기 확인**:
        ```bash
        flutter devices
        ```
    *   **앱 기동**:
        ```bash
        # 기본 디바이스로 실행
        flutter run

        # 크롬 브라우저 데모로 실행시
        flutter run -d chrome
        ```

## 3. 데이터베이스 마이그레이션 및 초기 데이터 설정

가상환경이 활성화된 상태에서 아래 명령어를 순서대로 실행하여 데이터베이스 스키마와 데이터를 최신 상태로 유지합니다.

### 3.1. Alembic DB 마이그레이션 적용
```bash
# 최신 DB 스키마 업데이트
alembic upgrade head

# 이전 스키마 버전으로 롤백이 필요한 경우
alembic downgrade -1
```

### 3.2. 기초 코드 및 테스트 데이터 로딩 (Seeding)
```bash
# 공통코드 및 기초 마스터 코드 설정
python seed_data.py

# 테스트용 시나리오 데이터 설정
python seed_scenario.py
```

## 4. 운영 서버 배포 Guide (Render.com & Supabase)

덤프링 백엔드 API 서버는 **Render**에 배포되어 있으며, GitHub `main` 브랜치에 코드가 푸시되면 자동으로 빌드 및 배포(CD)가 수행됩니다.

### 4.1. Render 웹 서비스 설정값
Render 대시보드에서 `New Web Service`를 생성하고 아래 정보를 세팅합니다.

*   **Name**: `dumpring-api`
*   **Region**: `Singapore (ap-southeast-1)`
*   **Branch**: `main`
*   **Runtime**: `Python`
*   **Build Command**: `pip install -r requirements.txt`
*   **Start Command**: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`

### 4.2. 환경 변수(Environment Variables) 설정
Render 서비스 관리 페이지의 `Environment` 탭에서 아래 세 개의 값을 등록합니다.
*   `DATABASE_URL`: `postgresql+asyncpg://postgres.vvdydhxmwdrrazwyjvvu:crypto.co.kr@aws-1-ap-northeast-2.pooler.supabase.com:6543/postgres?pgbouncer=true`
*   `PYTHON_VERSION`: `3.11.0`
*   `SECRET_KEY`: `[운영 환경 전용 암호화 키]`

> [!TIP]
> Render 서비스 사양에서 Supabase 연결 문자열 뒤에 `?pgbouncer=true` 매개변수를 추가하여 커넥션 풀을 더욱 효율적으로 사용하도록 처리해 두었습니다.

### 4.3. 수동 배포 트리거
GitHub 푸시 이외에 수동으로 즉시 배포를 갱신하려면:
*   Render Web Service 화면 우측 상단의 `Manual Deploy` -> `Clear build cache & deploy`를 클릭합니다.
