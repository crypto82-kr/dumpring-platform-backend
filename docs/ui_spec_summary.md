# 📱 덤프링(Dumpring) 통합 UI/UX 개발 명세서

본 문서는 덤프링 플랫폼의 모든 사용자 역할별 화면 구조, UI 구성 요소, 그리고 프론트엔드-백엔드 간 연동 규격을 정의합니다. IDE의 AI 개발 비서(Cursor, Windsurf, Copilot 등)에게 이 문서를 참고하여 컴포넌트와 페이지를 구현하도록 지시할 수 있습니다.

---

## 🎨 1. 공통 디자인 시스템 (Design System)
* **테마**: Light Mode 전용 (다크 모드 배제, 밝고 깨끗한 톤앤매너)
* **컬러 팔레트**:
  * **배경색 (Background)**: `#F9F8F6` (샌드 베이지) 및 `#FFFFFF` (순백색)
  * **주요 테마 색상 (Primary)**: `#F59E0B` (Mustard Yellow - 주요 버튼, 활성 상태)
  * **포인트 색상 (Accent)**: `#D97706` (Amber Orange - 강조 수치, 경고성 뱃지)
  * **메인 텍스트 (Text Primary)**: `#1F2937` (Charcoal Black)
  * **서브 텍스트 (Text Secondary)**: `#4B5563` (Slate Gray)
  * **카드 테두리 및 구분선**: `#E5E7EB` (Light Gray)
  * **정상/승인완료/성공 (Success)**: `#10B981` (Green)
  * **에러/반려/회차/취소 (Danger)**: `#EF4444` (Red)
* **타이포그래피**: `Malgun Gothic` (맑은 고딕) 또는 시스템 기본 Sans-serif 체 사용

## 🔑 2. 로그인 / 회원가입 흐름 (Authentication Flow)

### 화면 1: 통합 로그인 화면 (Login)
* **UI 요소**:
  * 상단: 덤프링 로고 및 슬로건
  * 중단: 휴대폰 번호 입력 필드, 비밀번호 입력 필드
  * 하단: `[로그인]` (머스터드 버튼), `[회원가입]` (텍스트 링크)

### 화면 2: 회원가입 - 역할 선택 (Role Selection)
* **UI 요소**:
  * "가입하실 회원 유형을 선택해 주세요" 안내 문구
  * 5개 카드 버튼 (라디오 버튼 방식, 머스터드 테두리 활성):
    1. 덤프 기사 (Driver)
    2. 덤프 차주 (Owner)
    3. 현장 관리자 (Site Manager)
    4. 현장 담당자 (Site Worker)
    5. 하차지 이용자 (Landowner)
  * 하단: `[다음 단계]` (비활성 그레이 -> 활성 머스터드)

### 화면 3: 회원가입 - 상세 정보 입력 (Details)
* **역할별 필수 제출 서류 및 필드**:
  * **기사 (Driver)**: 이름, 연락처, 운전면허증 사본 업로드 박스, `[가입 완료]` 버튼
  * **차주 (Owner)**: 이름, 연락처, 사업자등록증 사본 업로드, 정산용 통장 사본 업로드
  * **현장 관리자 (Manager)**: 소속 건설사명, 담당 현장명, 건설업 등록증 사본 업로드
  * **현장 담당자 (Worker)**: 소속 현장 검색 및 매핑 선택 필드 (소장의 승인 대기 상태로 가입)
  * **하차지 이용자 (Landowner)**: 사토장 주소, 토지 대장/허가증 사본 업로드

## 🚖 3. 덤프 기사 (Driver) 모바일 화면

### 화면 1: 메인 홈 & 배차 목록 (Job Board)
* **UI 요소**:
  * 상단: 기사 이름 및 현재 상태 (대기중 🟢 / 운행중 🟠)
  * 중단: 실시간 배차 목록 피드 (필터: 15톤/25톤, 상차지-하차지, 1회당 단가)
  * 동작: 배차 카드 클릭 시 상세 지도 및 수락 화면으로 이동
  * 하단: `[정산/완료 목록]` 탭 버튼

### 화면 2: 배차 수락 및 운행 상세 (Order Details)
* **UI 요소**:
  * 상단: 출발지(상차지) 및 도착지(하차지) 정보, 단가 (예: 150,000원)
  * 중단: 카카오맵/T맵 지도 API 영역 (상차지에서 하차지까지의 예상 궤적)
  * 하단: `[배차 수락 및 상차지 출발]` 머스터드 버튼

### 화면 3: 실시간 미터기 & 운행 중 (GPS Fare Meter)
* **UI 요소**:
  * 상단: 현재 목적지 정보 및 실시간 주행 거리/시간
  * 중단: **실시간 GPS 미터기 서클 위젯** (주행 속도, 경과 시간, 예상 정산 요금 실시간 증가 표시)
  * 하단: `[하차지 게이트 진입 완료]` 버튼 (하차지 반경 200m 진입 시 활성화)

## 🚚 4. 덤프 차주 (Owner) 모바일/웹 화면

### 화면 1: 차주 홈 대시보드 (Dashboard)
* **UI 요소**:
  * 상단: 소속 기사 현황 (총 N명, 운행 중인 기사 수), 보유 트럭 현황
  * 중단: 누적 정산 대기 요금 (예: 1,800,000원) 강조 표시
  * 하단: **실시간 기사-차량 연동 매핑 관제 목록** (번호판, 기사명, 현재 주행 단계)

### 화면 2: 소속 기사 관리 및 초대 (Driver Management)
* **UI 요소**:
  * 상단: 기사 휴대폰 번호 기입 필드 & `[초대 링크 발송 (SMS)]` 버튼
  * 하단: 소속 기사 목록 (카드 형태). 각 카드마다 배정 차량 지정용 **드롭다운 메뉴** (`차량 배정: 경기 80사 1234 ▼`) 제공

### 화면 3: 보유 차량 등록 (Vehicle Registry)
* **UI 요소**:
  * 차량 번호판 입력 필드
  * 덤프 규격 토글: `[15톤 덤프]` / `[25톤 덤프 (Selected)]`
  * 증빙 서류: 자동차등록증 사본 업로드 영역
  * 하단: `[신규 차량 등록 신청]` 머스터드 버튼

---

## 🏢 5. 현장 관리자 (Site Manager) 웹 콘솔

### 화면 1: 현장 종합 관제 대시보드 (Web Console)
* **UI 요소**:
  * 상단: 통계 지표 카드 3종 (운행 트럭 수, 총 운송 횟수, 누적 발생 운임)
  * 중단: 시간대별 배차 물량 통계 차트 (가로형/세로형 막대그래프)
  * 하단: 실시간 출입 덤프트럭 관제 테이블 (차량번호, 규격, 기사명, 실시간 진행 상태)

### 화면 2: B2B 매칭 오더 발행 (Create Order)
* **UI 요소**:
  * 오더 제목 입력, 모집 차량 대수 지정
  * 반출 토사 종류 선택 토글: `[양질토 (Selected)]`, `[뻘흙]`, `[암버럭]`
  * 연동할 하차지(사토장) 검색 및 1회당 지급 운송 단가 입력 필드
  * 하단: `[B2B 오더 발행 신청]` 머스터드 버튼

### 화면 3: 현장 직원 승인 및 권한 관리 (Worker Management)
* **UI 요소**:
  * 상단: 가입 신청 후 승인을 대기 중인 직원 목록 (이름, 연락처, `[소속 승인]` / `[반려]` 버튼 세트)
  * 하단: 현재 활성화되어 게이트 통제 권한을 가진 직원 목록

## 👷 6. 현장 담당자 (Site Worker) 모바일 화면

### 화면 1: 게이트 통제 홈 (Gate Control)
* **UI 요소**:
  * 상단: 3단 현황판 (입차대기 2대, 상차중 3대, 상차완료 18대)
  * 중단: 실시간 진입 대기 트럭 큐 리스트 (차량번호, 기사명, 자동/수동 진입 구분)
  * 하단: `[수동 체크인 / QR 스캔]` 플로팅 버튼

### 화면 2: 수동 입출차 승인 (Manual Checkout)
* **UI 요소**:
  * 승인 대상 차량 정보 (차량번호, 기사명, 톤수)
  * 반출 토사 종류 체크 토글: `[양질토]`, `[뻘흙]`, `[암버럭]`
  * 하단: `[상차 승인 (출발 처리)]` 머스터드 버튼 (터치 시 기사 단말기 미터기 시작)

---

## 🚜 7. 하차지 이용자 / 지주 (Landowner) 모바일 화면

### 화면 1: 사토장 대시보드 (Landowner Home)
* **UI 요소**:
  * 상단: **사토장 수용 한도 게이지** (65% 반입 완료, 잔여 수용량 표시 바)
  * 중단: 금일 반입 차량 수 및 누적 수금/정산액 요약 카드
  * 하단: 하차지 인근 지오펜싱(200m) 영역 안으로 진입한 실시간 트럭 목록

### 화면 2: 반입 승인 및 토질 검사 (Soil Verification)
* **UI 요소**:
  * 진입 트럭 상세 정보 (차량번호, 기사명, 선언 토사)
  * 육안 검토 체크박스 (토질 적합 검토, 유해물질 없음 확인)
  * 하단 제어 버튼:
    * `[최종 반입 승인]` (🟢 녹색 버튼 - 클릭 시 정산 확정 및 전표 발행)
    * `[반입 거부 및 회차 지시]` (🔴 적색 버튼 - 회차 처리 및 분쟁 접수)
  * 회차 선택 시: 거부 사유 입력 및 현장 증빙 사진 촬영 첨부 박스 활성화
